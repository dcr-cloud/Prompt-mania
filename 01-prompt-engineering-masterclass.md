## 🧠 PART 1: WHY PROMPTS FAIL (The Science)

AI models work on **probability + pattern matching**. When you give a prompt, 3 things happen:

1. **Priming** — Role dene se AI us expertise ka pattern activate karta hai
2. **Grounding** — Context dene se AI relevant knowledge select karta hai
3. **Steering** — Clear instructions se AI output shape hota hai

**Mental Model:**
```
Generic Prompt    = 40% relevant output
Structured Prompt = 85% relevant output
Layered Prompt    = 95%+ relevant output
```

---

## 🏗️ THE 7-LAYER PROMPT ARCHITECTURE

```
LAYER 1: ROLE        → Who is the AI?
LAYER 2: CONTEXT     → What's the situation?
LAYER 3: TASK        → What needs to be done?
LAYER 4: FORMAT      → How should output look?
LAYER 5: CONSTRAINTS → What to avoid/include?
LAYER 6: EXAMPLES    → Show, don't just tell
LAYER 7: VALIDATION  → How to self-check?
```

**Quality by Layers:**
```
1 Layer  = 30% quality
3 Layers = 60% quality
5 Layers = 80% quality
7 Layers = 95%+ quality
```

## ❌ THE 5 DEADLY SINS OF PROMPT WRITING

**SIN 1: Ambiguity**
- ❌ "Make it better"
- ✅ "Increase specificity by adding 2-3 concrete examples per concept"

**SIN 2: Assuming Context**
- ❌ "Summarize this"
- ✅ "Summarize this 30-min Huberman podcast on sleep. I'm a student who needs actionable protocols, not research details."

**SIN 3: Overloading Tasks**
- ❌ "Extract insights, create slides, suggest visuals, and write speaker notes"
- ✅ "First, extract 5 key insights. Then I'll ask you to convert them to slides."

**SIN 4: No Output Structure**
- ❌ "Give me the main ideas"
- ✅ Use the format template below

**SIN 5: No Quality Criteria**
- ❌ "Create slides from this"
- ✅ Define what "good" looks like with a checklist

## 🏆 THE TOP 1% PROMPT TEMPLATE

Copy and fill in the `[BRACKETS]`:

```
# ROLE & EXPERTISE
You are a dual-expert system combining:

Expert 1: [Content Expert]
- Specialization: [Specific domain]
- Approach: [How they think]

Expert 2: Learning Experience Designer
- Specialization: Creating "sticky" educational materials
- Approach: Make complex info memorable and actionable

# CONTEXT
- Source: [Video/Podcast/Article name and length]
- My audience: [Who will see these slides]
- My goal: [What I want to achieve]
- Success metric: [How I'll know it worked]

# TASK DECOMPOSITION

PHASE 1: Content Analysis
- Watch for: Repeated concepts (mentioned 3+ times)
- Extract: Core mental models, not just facts
- Filter out: Storytelling without lessons

PHASE 2: Insight Structuring
- Organize by: Concept hierarchy (foundational → advanced)
- For each concept capture:
  * What it is (definition)
  * Why it matters (significance)
  * How to apply (action)

PHASE 3: Slide Conversion
- Create [NUMBER] slides with this flow:
  * Slide 1: Context (What problem does this solve?)
  * Slides 2-4: Foundational concepts
  * Slides 5-8: Application concepts
  * Slide 9: Integration (how concepts connect)
  * Slide 10: Action plan

# FORMAT (Use EXACT template for each slide)
---
SLIDE [NUMBER]: [THEME]

HEADLINE: [Action-oriented, 6-9 words]

CORE MESSAGE: [One sentence, max 20 words]

SUPPORTING ELEMENTS: [Choose ONE]
Option A — Concept Breakdown:
  • Point 1: [Max 10 words]
  • Point 2: [Max 10 words]
  • Point 3: [Max 10 words]

Option B — Evidence-Based:
  • DATA: [Specific number/stat]
  • INTERPRETATION: [What it means]
  • IMPLICATION: [Why it matters]

VISUAL SUGGESTION: [Specific type — e.g., "3-column comparison table"]

KEY TAKEAWAY: [One sentence that could be tweeted]

RETENTION TEST: [One question to test understanding]
---

# CONSTRAINTS (Non-Negotiable)
✓ ONLY use concepts explicitly in the source
✓ EVERY claim backed by evidence
✓ ZERO generic advice
✓ Headlines: 6-9 words
✓ Bullets: Max 10 words each
✓ Each slide must teach something NON-OBVIOUS

# SELF-VALIDATION (Run before submitting)
□ Is concept explicitly in the source?
□ Is evidence specific?
□ Is this non-obvious?
□ Is headline 6-9 words?
□ Billboard Test: Understood in 6 seconds?
□ Tweet Test: Takeaway fits in 280 characters?
```

## 💡 PRO TIPS

- **Always give examples** of good vs bad output
- **Break complex tasks** into phases
- **Add a validation checklist** at the end
- **Specify exact word counts** for headlines and bullets
- **Name the exact video/source** — not just "this video"
